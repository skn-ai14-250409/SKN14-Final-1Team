인증

사용자가 Google 계정을 사용하여 앱에 로그인하게 합니다.

Google SDK로 로그인

기기 또는 브라우저에서 Google에 로그인한 사용자는 앱 또는 사이트에서 신속한 인증을 받을 수 있습니다. 기존 사용자는 자동으로 또는 탭/클릭 한 번으로 로그인합니다. 사용자가 탭 또는 클릭 한 번으로 새 계정을 만들 수 있는 옵션도 제공됩니다.

웹용 Google 계정으로 로그인 (원탭 사용) [https://developers.google.com/identity/gsi/web/guides/display-google-one-tap?hl=ko]

Android용 인증 관리자 [https://developers.google.com/identity/android-credential-manager?hl=ko]

iOS 및 macOS용 Google 로그인 [https://developers.google.com/identity/sign-in/ios/start?hl=ko]

우수사례 [https://developers.google.com/identity/sign-in/case-studies?hl=ko]

업계 표준

오픈소스 및 업계 표준 인증 패스키는 비밀번호를 대체하는 더 안전하고 쉬운 방식입니다. Google의 OAuth 2.0 API는 OpenID Connect 사양을 준수하고, OpenID 인증을 받았으며, 인증과 승인에 모두 사용할 수 있습니다.

패스키 [https://developers.google.com/identity/passkeys?hl=ko]

OpenID Connect [https://developers.google.com/identity/protocols/oauth2/openid-connect?hl=ko]

기존 로그인

이러한 API는 참고용으로 여기에 나열되어 있습니다.

Android용 원탭 [https://developers.google.com/identity/one-tap/android/legacy-get-started?hl=ko]

Android용 Google 로그인 [https://developers.google.com/identity/sign-in/android/legacy-gsi-start?hl=ko]

웹용 Google 로그인 [https://developers.google.com/identity/sign-in/web/sign-in?hl=ko]

승인

앱이 Google API 및 데이터를 사용하도록 승인합니다.

Google API 호출

OAuth 2.0 및 클라이언트 라이브러리 [https://developers.google.com/identity/protocols/oauth2?hl=ko#libraries]를 사용하여 빠르고 안전하게 Google API를 호출합니다. Google에서는 웹 서버, 클라이언트 측, 설치된 기기, 입력이 제한된 기기 애플리케이션과 같은 일반적인 OAuth 2.0 시나리오를 지원합니다. 앱을 인증 받고 프로덕션용으로 준비 [https://developers.google.com/identity/protocols/oauth2/production-readiness/policy-compliance?hl=ko]합니다.

Android 승인 [https://developers.google.com/identity/authorization/android?hl=ko]

웹 승인 [https://developers.google.com/identity/oauth2/web/guides/overview?hl=ko]

iOS/MacOS 승인 [https://developers.google.com/identity/sign-in/ios/api-access?hl=ko]

OAuth 2.0 사용 [https://developers.google.com/identity/protocols/oauth2?hl=ko]

Google 앱 및 기기와 데이터 공유

서비스 및 API를 Google과 통합하고 Google 어시스턴트, 스마트 홈, YouTube 등에서 미디어 및 데이터를 공유할 수 있습니다. 사용자 동의를 얻은 후 OAuth 2.0 표준 흐름을 통해 개별 Google 계정을 플랫폼의 계정과 안전하게 연결합니다. 맞춤 범위를 사용하여 사용자 개인 정보 보호를 강화하여 특정 사용 사례에 필요한 데이터만 공유하세요. Google의 데이터 사용 방식을 명확히 전달하여 사용자 신뢰를 높입니다.

Google 계정 연결 [https://developers.google.com/identity/account-linking?hl=ko]

사용자 인증 정보 관리

사용자 인증 정보를 안전하게 저장하고 가져옵니다.

Android용 인증 관리자

인증 관리자를 사용하여 비밀번호 없는 인증을 패스키와 원활하게 통합하여 앱의 사용자 환경을 개선하세요. 인증 관리자는 패스키, 비밀번호, 제휴 로그인 솔루션 (예: Google 계정으로 로그인)을 지원하는 Android Jetpack 라이브러리입니다.

Android용 인증 관리자 [https://developers.google.com/identity/android-credential-manager?hl=ko]

Android용 Blockstore

Android용 Blockstore API를 사용하면 앱이 사용자 비밀번호 저장과 관련된 복잡성이나 보안 위험 없이 사용자 인증 정보를 저장할 수 있습니다.
더 강력한 보안, 개선된 사용자 환경 및 기타 개선사항을 이용하려면 대신 인증 관리자에서 패스키 [https://developers.google.com/identity/passkeys?hl=ko]를 사용하는 것이 좋습니다.

Android 블록 스토어 [https://developers.google.com/identity/blockstore/android?hl=ko]

Android 자동 완성

앱의 양식에서 기기의 자동 완성 유틸리티를 사용하도록 설정합니다.

Android 자동 완성 프레임워크 [https://developer.android.com/guide/topics/text/autofill?hl=ko]

웹용 자동 완성

웹페이지의 양식에서 브라우저의 자동 완성 유틸리티를 사용하도록 설정합니다.

웹용 자동 완성 [https://web.dev/sign-in-form-best-practices/?hl=ko]

사용자 인증 정보 확인

확인된 사용자 인증 정보로 다중 인증(MFA)을 사용 설정합니다.

Android에서 사용자 인증

SMS Retriever API를 사용하면 인증 코드를 수동으로 입력할 필요 없이 SMS로 사용자를 확인할 수 있습니다.

SMS 리트리버 [https://developers.google.com/identity/sms-retriever/overview?hl=ko]

전화번호 힌트 [https://developers.google.com/identity/phone-number-hint/android?hl=ko]

웹에서 사용자 인증

WebOTP API를 사용하여 웹에서 전화번호를 인증하고 SMS를 통해 받은 일회용 비밀번호로 사용자를 지원합니다.

WebOTP API [https://web.dev/web-otp/?hl=ko]

관련 솔루션

[https://firebase.google.com/docs/auth/?hl=ko]

Firebase Authentication [https://firebase.google.com/docs/auth/?hl=ko]

Firebase 인증을 사용하면 모든 기기에서 사용자의 로그인과 온보딩을 통해 안전한 인증을 손쉽게 구축할 수 있습니다. 사용하기 쉬운 클라이언트 SDK와 함께 사용자를 안전하게 인증하는 백엔드 서비스를 제공합니다. 비밀번호 및 제휴 ID 공급업체 사용자 인증 정보를 사용하여 사용자를 인증할 수 있습니다. 또한 Firebase 인증은 앱에서 완전한 인증 환경을 구현할 수 있는 UI 라이브러리를 제공합니다.

자세히 알아보기 [https://firebase.google.com/docs/auth/?hl=ko]

[https://cloud.google.com/identity-platform?hl=ko]

Google Cloud Identity Platform [https://cloud.google.com/identity-platform?hl=ko]

Identity Platform은 고객 ID 및 액세스 관리(CIAM) 플랫폼으로, Google Cloud에서 앱에 ID 및 액세스 관리 기능을 추가하고 사용자 계정을 보호하며 안심하고 확장할 수 있도록 도와줍니다.

자세히 알아보기 [https://cloud.google.com/identity-platform?hl=ko]
