스타일 업데이트 테스트
프로덕션에서 사용 중인 스타일을 업데이트하려면 스타일 업데이트를 먼저 테스트해야 할 수도 있습니다.
참고: 업데이트된 기존 스타일을 확인하려면 업데이트된 지도 스타일 확인 [https://developers.google.com/maps/documentation/ios-sdk/cloud-customization/update-legacy?hl=ko#check-style]을 참고하세요.
프로덕션 지도 스타일을 복제합니다.
복제된 지도 스타일에서 원하는 대로 변경하고 게시합니다.
참고: 복제된 스타일이 방금 업데이트된 경우 업데이트된 스타일이 예상대로 표시되는지 확인하세요. 맞춤 스타일이 있는 지도 지형지물 옆에는 파란색 단색 점이 표시됩니다. 모든 지도 지형지물을 펼쳐 모든 변경사항을 찾습니다.
복제된 지도 스타일을 스테이징 환경에서 스타일을 테스트하는 데 사용할 수 있는 다른 지도 ID와 연결합니다.
스테이징 환경에서 스타일 변경사항을 테스트합니다.
결과가 만족스러우면 복제된 지도 스타일을 프로덕션 지도 ID와 연결합니다.
참고: 복제된 지도 스타일에는 복제한 스타일의 버전이 포함되지 않습니다. 스타일의 기록을 보관하려면 복제된 지도 스타일에서 테스트한 변경사항을 대신 가져와 이를 원본 지도 스타일로 설정할 수 있습니다.
참고: 스타일 변경사항이 앱 및 웹사이트에 반영되기까지는 몇 시간이 걸릴 수 있으며, 앱의 경우 기기가 인터넷에 연결되어 있지 않으면 시간이 더 걸릴 수 있습니다. 몇 시간 후에도 스타일 변경사항이 표시되지 않으면 스타일 변경사항이 업데이트되지 않음 [https://developers.google.com/maps/documentation/ios-sdk/cloud-customization/troubleshoot?hl=ko#style-delay]을 참고하세요.
