도움말 및 알려진 제한사항
bookmark_border
Gmail에서 지원하는 모든 플랫폼과 브라우저에서 AMP 이메일이 올바르고 일관되게 렌더링되도록 하려면 다음 도움말을 따르세요.
참고: 이 목록은 여러 이메일 제공업체에 적용되는 이메일용 AMP 권장사항 [https://amp.dev/documentation/guides-and-tutorials/develop/amp_email_best_practices?format=email]에 추가된 목록입니다.
받은편지함에서 이메일을 볼 때 제목 옆에 표시되는 요약 텍스트 (preheader)는 이메일의 text/html 또는 text/plain 부분을 기반으로 하며 text/x-amp-html 부분은 고려하지 않습니다.
Gmail의 검색 기능은 이메일의 동적 부분에 색인을 생성하지 않습니다. 사용자가 이메일을 쉽게 찾을 수 있도록 이메일의 text/html 또는 text/plain 부분에 가장 중요한 키워드를 포함하세요.
이메일 마크업 [https://developers.google.com/workspace/gmail/markup?hl=ko] 및 프로모션 주석 [https://developers.google.com/workspace/gmail/promotab?hl=ko]은 이메일의 text/html 부분에서 파싱되고 text/x-amp-html 부분에서는 무시됩니다.
엔드포인트로 전송되는 amp-form 양식 데이터는 <form> 요소에 설정된 enctype 속성 값과 관계없이 항상 application/x-www-form-urlencoded로 인코딩됩니다.
동적 이메일에는 최신 콘텐츠가 포함되므로 사용자는 동일한 대화목록 내에 중복 이메일을 수신할 수 있습니다. 예를 들어 동일한 댓글 대화목록에 대한 하나의 이메일 대화목록에 있는 Google Docs 댓글 이메일은 모두 펼치면 동일하게 보입니다. Gmail은 중복 콘텐츠가 포함된 이메일을 접어 긴 대화목록에 이러한 중복 이메일이 표시되지 않도록 할 수 있습니다. 이메일이 중복으로 인식되도록 하려면 AMP 마크업에 email.contentIds 메타 태그를 추가합니다.
<head>
  ...
  <meta name="email.contentIds" content="id1,id2,id3">
  ...
</head>
이 메타 태그의 콘텐츠에는 동적 이메일 내 고유한 콘텐츠를 식별하는 문자열의 쉼표로 구분된 목록이 포함되어야 합니다. 대화목록의 마지막 이메일의 콘텐츠 ID 세트가 대화목록의 다른 모든 이메일의 콘텐츠 ID 세트의 (비엄격한) 상위 집합인 경우 마지막 이메일을 제외한 모든 이메일이 적절하게 접힙니다.
예를 들어 다음과 같은 콘텐츠 ID 세트가 포함된 이메일 대화목록은 접힙니다.
id1
id1
id1
다음 Content ID 세트가 포함된 이메일 대화목록도 접힙니다.
id1, id2
id2, id3
id1, id2, id3
이메일에 입력 요소가 있는 양식이 포함되어 있는 경우 입력이 변경되면 사용자가 양식을 제출하지 않고 이메일에서 나가기 전에 확인 메시지를 볼 수 있습니다. 사용자가 양식을 지우거나 메시지를 삭제하는 경우와 같이 이 확인 메시지가 표시되지 않는 경우도 있습니다.
amp-mustache 템플릿에는 세트 구분자 태그를 포함할 수 없습니다.
