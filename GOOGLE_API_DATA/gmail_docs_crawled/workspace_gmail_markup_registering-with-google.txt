작업 / 스키마 가이드라인
이메일에서 스키마를 어떻게 사용할지 기대됩니다. 오늘 자체 통합 테스트를 시작할 수 있습니다. x@gmail.com에서 x@gmail.com으로 보내는 모든 스키마는 Google 제품에 표시됩니다. 지금 바로 사용해 보세요.
마크업된 이메일을 사용자에게 전송할 준비가 되면 Google에 등록해야 합니다. 다음 절차를 따르세요.
아래에 나열된 모든 가이드라인과 요구사항을 충족하는지 확인하세요.
마크업/스키마를 포함하여 프로덕션 서버 (또는 유사한 DKIM/SPF/From:/Return-Path: 헤더가 있는 서버)에서 실제 이메일을 schema.whitelisting+sample@gmail.com [mailto:schema.whitelisting+sample@gmail.com]으로 보냅니다. 이는 등록 가이드라인 [https://developers.google.com/workspace/gmail/markup/registering-with-google?hl=ko#registration_guidelines]에 나열된 모든 가이드라인과 요구사항을 준수하는지 확인하기 위해 필요합니다.
테스트/빈 이메일을 보내거나 스키마가 포함되지 않은 이메일을 보내거나 검토를 위해 이메일을 보내지 않으면 애플리케이션이 자동으로 삭제됩니다.
이메일을 보내기 전에 마크업이 올바른지 확인하세요. 자세한 내용은 스키마 테스트 [https://developers.google.com/workspace/gmail/markup/testing-your-schema?hl=ko]를 참고하세요. 특히 이메일이 이메일 마크업 테스터 [https://www.google.com/webmasters/markup-tester/?hl=ko]를 통과하고 오류가 없는지 확인하고 가능한 한 많은 데이터를 포함해야 합니다.
Gmail은 이메일을 전달할 때 모든 마크업을 삭제합니다. 이메일을 전달하지 말고 직접 보내세요.
등록 양식 [https://docs.google.com/a/google.com/forms/d/e/1FAIpQLSfT5F1VJXtBjGw2mLxY2aX557ctPTsCrJpURiKJjYeVrugHBQ/viewform?pli=1&hl=ko]을 작성해 주시면 연락드리겠습니다.

전송한 스키마가 처리되려면 다음 가이드라인을 충족해야 합니다.
이메일 발신자 품질 가이드라인
이메일이 DKIM 또는 SPF를 통해 인증 [https://support.google.com/mail/answer/180707?hl=ko]되어야 합니다.
SPF 검사 또는 DKIM 서명의 최상위 도메인 (TLD)이 From: 이메일 주소의 TLD와 일치해야 합니다.
예를 들어 From: foo@bar.com을 사용하는 경우 DKIM 또는 SPF는 bar.com 또는 sub.bar.com에 대한 것이어야 합니다.
Gmail은 Return-Path: 이메일의 도메인에 대해서만 SPF를 확인합니다. DKIM이 아닌 SPF만 사용하는 경우 Return-Path: 이메일이 From: 이메일의 TLD와 일치해야 합니다.
SPF 또는 DKIM 설정이 올바른지 신속하게 확인하려면 이메일 인증 도움말 [https://support.google.com/mail/answer/180707?hl=ko]을 참고하세요.
이메일은 고정 이메일 주소(예: foo@bar.com)에서 발송해야 합니다.
이메일은 Gmail 사용자에게 보낸 메일이 차단되거나 스팸으로 분류되는 것을 방지하기 [https://support.google.com/mail/answer/81126?hl=ko] 가이드라인을 따라야 합니다.
도메인에서 최소 몇 주 동안 매일 Gmail로 수백 통 이상의 메일을 꾸준히 전송한 기록이 있습니다.
사용자의 스팸 신고율이 매우 낮습니다.
작업 / 스키마 가이드라인
사용 가능한 가장 높은 충실도의 동작을 사용해야 합니다. 예를 들어 인앱 액션 (원클릭, 답장, 리뷰)으로 상호작용을 달성할 수 있는 경우 이를 사용해야 합니다. 더 복잡한 상호작용의 경우 바로가기 작업을 사용할 수 있습니다.
작업은 높은 상호작용률이 예상되는 거래 메일에 사용해야 합니다. 프로모션 대량 메일에는 사용하면 안 됩니다.
바로가기 작업:
작업을 실행할 수 있는 특정 페이지로 딥 링크되어야 합니다.
버튼 라벨은 취해야 할 명확한 조치를 반영해야 하며 사용자가 이동할 페이지와 일치해야 합니다.
작업 라벨에 구두점이나 전체 대문자를 포함해서는 안 됩니다. 짧고 간결해야 합니다.
사용 사례에 인앱 작업을 사용할 수 있는 경우 '이메일 주소 확인' 또는 '검토해 주세요'와 같이 인앱 작업을 사용해야 하며 이동 작업은 사용하면 안 됩니다.
현재는 상호작용률이 높은 매우 구체적인 고가치 사용 사례 (예: 항공편 체크인, 배송 추적 링크)에 대해서만 이동 작업을 승인하고 있습니다.
