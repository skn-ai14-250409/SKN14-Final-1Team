이 API 참조는 리소스 유형을 기준으로 구성되어 있습니다. 각 리소스 유형은 하나 이상의 데이터 표현 및 하나 이상의 메소드를 갖습니다.
리소스 유형
Acl
Acl 리소스에 대한 자세한 내용은 리소스 표현 [https://developers.google.com/workspace/calendar/api/v3/reference/acl?hl=ko#resource] 페이지를 참고하세요.
메서드 HTTP 요청 설명
별도로 명시하지 않는 한 URI는 https://www.googleapis.com/calendar/v3을 기준으로 합니다.
삭제 [https://developers.google.com/workspace/calendar/api/v3/reference/acl/delete?hl=ko] DELETE  /calendars/calendarId/acl/ruleId 액세스 제어 규칙을 삭제합니다.
get [https://developers.google.com/workspace/calendar/api/v3/reference/acl/get?hl=ko] GET  /calendars/calendarId/acl/ruleId 액세스 제어 규칙을 반환합니다.
insert [https://developers.google.com/workspace/calendar/api/v3/reference/acl/insert?hl=ko] POST  /calendars/calendarId/acl 액세스 제어 규칙을 만듭니다.
list [https://developers.google.com/workspace/calendar/api/v3/reference/acl/list?hl=ko] GET  /calendars/calendarId/acl 캘린더의 액세스 제어 목록에 있는 규칙을 반환합니다.
patch [https://developers.google.com/workspace/calendar/api/v3/reference/acl/patch?hl=ko] PATCH  /calendars/calendarId/acl/ruleId 액세스 제어 규칙을 업데이트합니다. 이 메소드는 패치 의미 체계를 지원합니다. 각 패치 요청은 3개의 할당량 단위를 사용합니다. get 뒤에 update을 사용하는 것이 좋습니다. 지정하는 필드 값은 기존 값을 대체합니다. 요청에 지정하지 않은 필드는 변경되지 않고 유지됩니다. 배열 필드(지정된 경우)는 기존 배열을 덮어씁니다. 이렇게 하면 이전 배열 요소가 삭제됩니다.
업데이트 [https://developers.google.com/workspace/calendar/api/v3/reference/acl/update?hl=ko] PUT  /calendars/calendarId/acl/ruleId 액세스 제어 규칙을 업데이트합니다.
시청 [https://developers.google.com/workspace/calendar/api/v3/reference/acl/watch?hl=ko] POST  /calendars/calendarId/acl/watch ACL 리소스의 변경사항을 확인합니다.

list [https://developers.google.com/workspace/calendar/api/v3/reference/calendarList/list?hl=ko] GET  /users/me/calendarList 사용자의 캘린더 목록에 있는 캘린더를 반환합니다.

시청 [https://developers.google.com/workspace/calendar/api/v3/reference/calendarList/watch?hl=ko] POST  /users/me/calendarList/watch CalendarList 리소스의 변경사항을 확인합니다.
캘린더
Calendars 리소스에 대한 자세한 내용은 리소스 표현 [https://developers.google.com/workspace/calendar/api/v3/reference/calendars?hl=ko#resource] 페이지를 참고하세요.

별도로 명시하지 않는 한, URI는 https://www.googleapis.com/calendar/v3을 기준으로 합니다.
지우기 [https://developers.google.com/workspace/calendar/api/v3/reference/calendars/clear?hl=ko] POST  /calendars/calendarId/clear 기본 캘린더를 삭제합니다. 이 작업을 수행하면 계정의 기본 캘린더와 연결된 모든 일정이 삭제됩니다.
삭제 [https://developers.google.com/workspace/calendar/api/v3/reference/calendars/delete?hl=ko] DELETE  /calendars/calendarId 보조 캘린더를 삭제합니다. 기본 캘린더의 모든 일정을 삭제하려면 calendars.clear를 사용하세요.
get [https://developers.google.com/workspace/calendar/api/v3/reference/calendars/get?hl=ko] GET  /calendars/calendarId 캘린더의 메타데이터를 반환합니다.
insert [https://developers.google.com/workspace/calendar/api/v3/reference/calendars/insert?hl=ko] POST  /calendars 보조 캘린더를 만듭니다.
patch [https://developers.google.com/workspace/calendar/api/v3/reference/calendars/patch?hl=ko] PATCH  /calendars/calendarId 캘린더의 메타데이터를 업데이트합니다. 이 메소드는 패치 의미 체계를 지원합니다. 각 패치 요청은 3개의 할당량 단위를 사용합니다. get 뒤에 update을 사용하는 것이 좋습니다. 지정하는 필드 값은 기존 값을 대체합니다. 요청에 지정하지 않은 필드는 변경되지 않고 유지됩니다. 배열 필드(지정된 경우)는 기존 배열을 덮어씁니다. 이렇게 하면 이전 배열 요소가 삭제됩니다.
업데이트 [https://developers.google.com/workspace/calendar/api/v3/reference/calendars/update?hl=ko] PUT  /calendars/calendarId 캘린더의 메타데이터를 업데이트합니다.

이벤트
Events 리소스에 대한 자세한 내용은 리소스 표현 [https://developers.google.com/workspace/calendar/api/v3/reference/events?hl=ko#resource] 페이지를 참고하세요.
메서드 HTTP 요청 설명
별도로 명시하지 않는 한, URI는 https://www.googleapis.com/calendar/v3을 기준으로 합니다.
삭제 [https://developers.google.com/workspace/calendar/api/v3/reference/events/delete?hl=ko] DELETE  /calendars/calendarId/events/eventId 일정을 삭제합니다.
get [https://developers.google.com/workspace/calendar/api/v3/reference/events/get?hl=ko] GET  /calendars/calendarId/events/eventId Google Calendar ID를 기반으로 일정을 반환합니다. iCalendar ID를 사용하여 일정을 검색하려면 iCalUID 매개변수를 사용하여 events.list 메서드 [https://developers.google.com/workspace/calendar/api/v3/reference/events/list?hl=ko#iCalUID]를 호출합니다.
import [https://developers.google.com/workspace/calendar/api/v3/reference/events/import?hl=ko] POST  /calendars/calendarId/events/import 일정을 가져옵니다. 이 작업은 기존 일정의 비공개 사본을 캘린더에 추가하는 데 사용됩니다. eventType이 default인 이벤트만 가져올 수 있습니다.
지원 중단된 동작: default가 아닌 이벤트를 가져오면 유형이 default로 변경되고 이벤트 유형별 속성이 있는 경우 모두 삭제됩니다.
insert [https://developers.google.com/workspace/calendar/api/v3/reference/events/insert?hl=ko] POST  /calendars/calendarId/events 일정을 만듭니다.
인스턴스 [https://developers.google.com/workspace/calendar/api/v3/reference/events/instances?hl=ko] GET  /calendars/calendarId/events/eventId/instances 지정된 반복 일정의 인스턴스를 반환합니다.
list [https://developers.google.com/workspace/calendar/api/v3/reference/events/list?hl=ko] GET  /calendars/calendarId/events 지정된 캘린더의 일정을 반환합니다.
이동 [https://developers.google.com/workspace/calendar/api/v3/reference/events/move?hl=ko] POST  /calendars/calendarId/events/eventId/move 일정을 다른 캘린더로 이동합니다. 즉, 일정의 주최자를 변경합니다. default 이벤트만 이동할 수 있습니다. birthday, focusTime, fromGmail, outOfOffice, workingLocation 이벤트는 이동할 수 없습니다.

quickAdd [https://developers.google.com/workspace/calendar/api/v3/reference/events/quickAdd?hl=ko] POST  /calendars/calendarId/events/quickAdd 간단한 텍스트 문자열을 기반으로 이벤트를 만듭니다.
필수 쿼리 매개변수: text
업데이트 [https://developers.google.com/workspace/calendar/api/v3/reference/events/update?hl=ko] PUT  /calendars/calendarId/events/eventId 일정을 업데이트합니다. 이 메서드는 패치 시맨틱스를 지원하지 않으며 항상 전체 이벤트 리소스를 업데이트합니다. 부분 업데이트를 수행하려면 get 다음에 etag를 사용하여 update를 실행하여 원자성을 보장합니다.
시청 [https://developers.google.com/workspace/calendar/api/v3/reference/events/watch?hl=ko] POST  /calendars/calendarId/events/watch 이벤트 리소스의 변경사항을 확인합니다.
