개요
Places Aggregate API는 위치, 유형, 운영 상태, 가격대, 사용자 평점과 같은 기준에 따라 지정된 지역 내 장소에 관한 유용한 정보를 제공하는 서비스입니다. 이 서비스를 사용하면 지정된 위치 주변의 특정 장소 유형 [https://developers.google.com/maps/documentation/places/web-service/place-types?hl=ko]의 밀도를 분석하고 '이 위치에서 반경 5km 이내에 있는 별점 5개짜리 $$$ 레스토랑은 무엇이며 몇 개인가?'와 같은 질문에 답할 수 있습니다.
결과는 집계된 수 또는 관심 검색 영역에서 식별된 장소 ID로 반환됩니다. 사용자는 Place Details API [https://developers.google.com/maps/documentation/places/web-service/place-details?hl=ko]를 사용하여 이러한 장소 ID에 관한 자세한 정보를 검색할 수 있습니다.
Places Aggregate API를 사용해야 하는 이유
Places Aggregate API를 사용하면 다양한 장소에 관해 제공되는 포괄적인 정보를 기반으로 사용자가 데이터 기반 결정을 내릴 수 있습니다. 정확하고 최신 장소 모델을 활용하여 다음과 같은 주요 사용 사례를 지원합니다.
비즈니스: 경쟁업체와 신규 지점의 잠재적 위치를 분석합니다.
개발자: 맞춤 추천을 제공하는 애플리케이션을 빌드합니다.
연구원: 특정 분야의 추세와 패턴을 살펴봅니다.
Places Aggregate API로 할 수 있는 작업
Places Aggregate API를 사용하면 다음 정보를 확인할 수 있습니다.
개수: 기준과 일치하는 장소의 수를 가져옵니다.
장소 ID: 기준에 맞는 특정 장소의 장소 ID [https://developers.google.com/maps/documentation/places/web-service/place-id?hl=ko]를 가져옵니다.
필터링을 사용하여 장소 유형, 영업시간, 가격대, 고객 평가와 같은 다양한 속성을 기준으로 검색을 세부적으로 조정할 수도 있습니다.
참고: 장소 ID는 count이 100 이하인 경우에만 반환됩니다.

새 카페를 오픈하는 경우
레스토랑 주인이 새 카페를 열고 싶어 합니다. 이를 위해 먼저 카페의 핫스팟을 시각화하여 밀도가 높고 낮은 지역을 파악하여 비즈니스 결정을 내리고자 합니다. Places Aggregate API를 사용하면 운영 상태, 가격대, 고객 평가와 같은 속성을 기반으로 특정 반경 내 카페 수를 분석하여 다음 매장을 열 위치에 관한 데이터 기반 결정을 내릴 수 있습니다.

부동산 투자 회사
부동산 투자 회사는 재무 모델을 개선하고 계획된 부동산 투자의 ROI를 정확하게 파악하려고 합니다. Places Aggregate API를 사용하면 ATM, 병원, 대중교통역, 식료품점 등 잠재적 투자 부동산 근처의 편의시설에 관한 자세한 데이터를 수집하여 잠재적 투자 부동산 근처의 편의시설을 파악할 수 있습니다.
소매 배송 서비스
새 도시로 확장할 때 소매 배송 서비스는 레스토랑, 편의점, 주류 판매점과 같은 인기 있는 소비자 목적지의 밀도를 기반으로 지역에 할당할 배송 기사의 수를 결정해야 합니다. 배달 서비스는 API를 사용하여 도시 전체의 이러한 업체의 총수를 파악하여 리소스를 효과적으로 계획하고 할당할 수 있습니다.

Places Aggregate API 작동 방식
Places Aggregate API를 사용하면 필터를 지정하여 검색 기준을 좁힐 수 있습니다. INSIGHT_COUNT 또는 INSIGHT_PLACES의 통계 유형을 선택한 후 다음을 비롯한 필터 기준을 추가할 수 있습니다.
위치: 원, 지역 또는 맞춤 다각형을 사용하여 관심분야를 정의합니다.
유형: 관심 있는 장소 유형을 지정합니다.
운영 상태: 운영 상태를 기준으로 장소를 필터링합니다.
가격대: 가격대를 기준으로 장소를 필터링합니다.
평점: 사용자 평점을 기준으로 장소를 필터링합니다.
API 응답 ComputeInsightsResponse [https://developers.google.com/maps/documentation/places-aggregate/reference/rest/v1/TopLevel/computeInsights?hl=ko#response-body] 객체에는 요청 통계 결과가 포함됩니다. 예를 들어 INSIGHT_COUNT를 선택한 경우 대답에 총 장소 수가 포함되고 INSIGHT_PLACES를 선택한 경우 대답에 장소 ID 목록이 포함됩니다.
Places Aggregate API 사용 방법
1 설정하기 Google Cloud 프로젝트 설정하기 [https://developers.google.com/maps/documentation/places-aggregate/cloud-setup?hl=ko]부터 시작하여 이어지는 안내를 완료합니다.
2 일치하는 장소의 개수를 가져오도록 요청합니다. 첫 번째 요청하기 [https://developers.google.com/maps/documentation/places-aggregate/make-your-first-request?hl=ko]를 참고하세요.
3 요청 매개변수에 대해 알아보기 요청 매개변수 [https://developers.google.com/maps/documentation/places-aggregate/request-parameters?hl=ko]를 참고하세요.
