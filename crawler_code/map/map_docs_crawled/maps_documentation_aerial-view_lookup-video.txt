의견 보내기
동영상 가져오기
유효한 주소 또는 동영상 ID가 주어지면 lookupVideo [https://developers.google.com/maps/documentation/aerial-view/reference/rest/v1/videos/lookupVideo?hl=ko] 메서드는 항공 뷰 API 동영상 집합을 식별하는 URI를 반환합니다. lookupVideo 메서드는 Google에서 해당 위치의 동영상을 이미 생성한 경우에만 URI를 반환합니다.
동영상이 있는지 확인하려면 lookupVideoMetadata [https://developers.google.com/maps/documentation/aerial-view/lookup-video-metadata?hl=ko] 메서드를 사용합니다.
참고: Google에서 특정 주소의 항공 동영상을 생성하면 누구나 해당 동영상을 사용할 수 있습니다. 즉, 동영상은 특정 사용자나 계정과 연결되지 않습니다.
API 사용 방법
URL 인코딩된 [https://developers.google.com/maps/url-encoding?hl=ko] 미국 우편 주소 또는 videoId [https://developers.google.com/maps/documentation/aerial-view/reference/rest/v1/VideoMetadata?hl=ko]를 전달하여 이 엔드포인트를 호출합니다.
curl -X GET "https://aerialview.googleapis.com/v1/videos:lookupVideo?key=YOUR_API_KEY&address=POSTAL_ADDRESS"
그러면 API가 다음 세 가지 응답 중 하나를 반환합니다.
동영상이 있는 경우 lookupVideo는 ACTIVE의 처리 상태와 지원되는 모든 미디어 유형의 가로 및 세로 URI, 메타데이터 [https://developers.google.com/maps/documentation/aerial-view/reference/rest/v1/VideoMetadata?hl=ko]를 반환합니다.
참고: 이 엔드포인트에 대한 호출은 동영상이 있고 동영상 URI가 반환되는 경우에만 청구됩니다.
동영상이 YouTube 시스템에 있지만 아직 완전히 렌더링되지 않은 경우 lookupVideo는 PROCESSING의 처리 상태와 videoId를 반환합니다. videoId를 사용하면 렌더링이 완료된 후 동영상을 가져올 수 있습니다.
동영상이 아직 존재하지 않으면 API는 404 상태 코드를 반환합니다. 이 경우 renderVideo [https://developers.google.com/maps/documentation/aerial-view/reference/rest/v1/videos/renderVideo?hl=ko]를 호출하여 새 동영상을 생성 [https://developers.google.com/maps/documentation/aerial-view/generate-video?hl=ko]할 수 있습니다.
반환된 동영상 URI를 사용하는 방법
주소당 한 번 lookupVideo 메서드를 호출하거나 뷰당 videoID를 호출합니다. 대답의 동영상 URI는 단기 URI라고 하며 짧은 시간이 지나면 만료됩니다.
Google에서는 동영상을 자주 업데이트합니다. 따라서 고객에게 최신 콘텐츠를 표시하려면 동영상을 표시할 때마다 lookupVideo 메서드를 호출하세요.
주의: 항공 뷰 동영상은 다운로드하거나 저장하거나 캐시할 수 없습니다. 자세한 내용은 Google Maps Platform 서비스 약관 [https://cloud.google.com/maps-platform/terms/?hl=ko]을 참고하세요.
접근성 지원 동영상
접근성을 지원하기 위해 고객에게 항공 뷰 동영상을 표시할 때 스크립트 또는 동영상 설명을 추가하는 것이 좋습니다. 이렇게 하면 스크린 리더가 사용자에게 스크립트 또는 설명을 알릴 수 있습니다.
예를 들어 다음과 같은 형식으로 설명을 추가합니다.
'Google 지도에서 제공하는 주소의 포토리얼리스틱 항공 뷰입니다.'
주소 대신 다음과 같은 형식으로 동영상 설명을 추가할 수 있습니다.
'이 이미지는 Google 지도에서 제공하는 뉴욕 엠파이어 스테이트 빌딩의 포토리얼리스틱 항공 뷰입니다.'
나중에 사용할 수 있도록 동영상 ID 저장
항공 뷰 API의 서비스별 약관에 따라 항공 뷰 API 동영상 ID에는 Google Maps Platform 서비스 약관의 3.2.3(b)(캐싱 금지)항 [https://cloud.google.com/maps-platform/terms/?hl=ko]에 명시된 캐싱 제한사항이 적용되지 않습니다. 따라서 나중에 사용하기 위해 동영상 ID 값을 저장할 수 있습니다.

렌더링 프로세스는 1시간에서 몇 시간까지 걸릴 수 있습니다. 동영상이 아직 처리 중인 경우 lookupVideo를 사용하여 동영상 상태가 ACTIVE 상태를 반환할 때까지 폴링할 수 있습니다.
폴링하려면 videoId을 전달하여 lookupVideo에 요청합니다. 렌더링에는 몇 시간이 걸릴 수 있으므로 지수 백오프 [https://developers.google.com/maps/documentation/aerial-view/web-api-best-practices?hl=ko#exponential-backoff]를 사용하여 호출 간격을 늘리세요.
videoId를 전달하는 요청 예시
lookupVideo를 호출하고 주소에 대한 항공 뷰 동영상이 없음을 알게 되면 해당 주소로 renderVideo를 호출할 수 있습니다. renderVideo은 videoId이 포함된 VideoMetadata [https://developers.google.com/maps/documentation/aerial-view/reference/rest/v1/VideoMetadata?hl=ko] 객체를 반환합니다. 나중에 주소 대신 videoId를 사용하여 lookupVideo에 전달하여 동영상을 가져올 수 있습니다.
다음 코드 예는 lookupVideo를 호출하여 videoId을 전달하는 방법을 보여줍니다.
curl -X GET "https://aerialview.googleapis.com/v1/videos:lookupVideo?key=YOUR_API_KEY&videoId=VIDEO_ID"
