Street View Static API 개요
유럽 경제 지역 (EEA) 개발자
시작하기 전에: Street View Static API를 사용하려면 결제 계정이 있고 Street View Static API가 사용 설정된 프로젝트가 필요합니다. 프로젝트 소유자 및 결제 관리자를 여러 명 만들어 이러한 역할을 맡은 담당자가 팀에 상주하도록 하는 것이 좋습니다. 자세한 내용은 Cloud 콘솔에서 설정하기 [https://developers.google.com/maps/documentation/streetview/cloud-setup?hl=ko]를 참고하세요.
Street View Static API는 JavaScript를 사용하지 않고 정적 (비대화형) 스트리트 뷰 파노라마 또는 썸네일을 웹페이지에 삽입합니다. 표준 HTTP 요청을 통해 전송된 URL 매개변수로 뷰포트를 정의합니다. 요청은 정적 이미지를 반환합니다.
https://maps.googleapis.com/maps/api/streetview?size=400x400&location=47.5763831,-122.4211769
&fov=80&heading=70&pitch=0&key=YOUR_API_KEY&signature=YOUR_SIGNATURE
필수 및 선택적 매개변수, 요청 및 응답 예시는 개발자 가이드 [https://developers.google.com/maps/documentation/streetview/request-streetview?hl=ko]를 참고하세요.
참고: 스트리트 뷰 이미지는 Google(개발자가 관리하는 비공개 이미지)과 공개(UGC, 공개 사용자 제작 콘텐츠)의 두 가지 소스 [https://www.google.com/streetview/?hl=ko]를 통해 제공됩니다. 한 가지 소스의 스트리트 뷰 이미지만 사용할 수는 없습니다. 스트리트 뷰 이미지 개인정보처리방침 [https://www.google.com/streetview/privacy/?hl=ko]에 대해 자세히 알아보세요.
모든 Street View Static API 요청에는 API 키가 필요하며 API 보안을 극대화하려면 디지털 서명이 있어야 합니다. 정적 웹 API 권장사항 [https://developers.google.com/maps/documentation/streetview/static-web-api-best-practices?hl=ko] 및 API 보안 권장사항 [https://developers.google.com/maps/api-security-best-practices?hl=ko]에 대한 Google의 권장사항을 알아보세요.
Street View Static API는 정적(대화형이 아닌) 스트리트 뷰 파노라마를 삽입하기 위한 각 요청에 대해 요금이 청구됩니다. 자세한 내용은 사용량 및 결제 [https://developers.google.com/maps/documentation/streetview/usage-and-billing?hl=ko]를 참고하세요.
모니터링 및 보고 [https://developers.google.com/maps/documentation/streetview/report-monitor?hl=ko]를 통해 API 사용량과 지출을 관리하세요.
추가 정보
Street View Static API 또는 기타 Google Maps API 제품 사용에 대한 자세한 내용은 Maps API 지원 페이지 [https://developers.google.com/maps/support?hl=ko]를 참고하세요.
360도 스트리트 뷰 이미지를 게시하려면 Streetview Publish API [https://developers.google.com/streetview/publish/?hl=ko]를 참고하세요.
