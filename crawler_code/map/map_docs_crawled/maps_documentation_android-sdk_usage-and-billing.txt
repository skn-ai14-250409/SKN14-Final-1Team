Android용 Maps SDK 사용량 및 결제
알림: Android용 Maps SDK를 사용하려면 각 프로젝트에서 결제를 사용 설정 [https://console.cloud.google.com/project/_/billing/enable?utm_source=Docs_BillingNudge&hl=ko]하고 모든 API 또는 SDK 요청에 API 키 [https://developers.google.com/maps/documentation/android-sdk/get-api-key?hl=ko] 또는 OAuth 토큰을 포함해야 합니다.
Android용 Maps SDK는 사용한 만큼만 지불하는 가격 모델 [https://developers.google.com/maps/documentation/android-sdk/usage-and-billing?hl=ko#new-payg]을 사용합니다. Android용 Maps SDK 요청은 다음 세 가지 SKU 중 하나를 호출합니다. SKU: 모바일 네이티브 동적 지도 [https://developers.google.com/maps/documentation/android-sdk/usage-and-billing?hl=ko#mobile-dynamic], SKU: 동적 지도 [https://developers.google.com/maps/documentation/android-sdk/usage-and-billing?hl=ko#dynamic-maps] 또는 SKU: 동적 스트리트 뷰 [https://developers.google.com/maps/documentation/android-sdk/usage-and-billing?hl=ko#dynamic-street-view]. 전체 Google 이용약관과 함께 Android용 Maps SDK와 관련된 사용량 한도 [https://developers.google.com/maps/documentation/android-sdk/usage-and-billing?hl=ko#usage-limits]가 적용됩니다. Google Cloud 콘솔에서 제공되는 도구를 사용하여 비용 및 사용량을 관리 [https://developers.google.com/maps/documentation/android-sdk/usage-and-billing?hl=ko#set-caps]하세요.
Android용 Maps SDK 요금 청구 방식
Android용 Maps SDK는 사용한 만큼만 지불하는 가격 모델을 사용합니다. Google Maps Platform API 및 SDK는 SKU를 기준으로 청구됩니다. 사용량은 각 SKU에 따라 추적되며 모든 API 또는 SDK에 둘 이상의 제품 SKU [https://developers.google.com/maps/billing-and-pricing/sku-mapping-old-to-new?hl=ko]가 있을 수 있습니다. 비용은 다음과 같이 계산됩니다.
SKU 사용량 × 사용당 가격
가격 및 사용량 계산기 [https://cloud.google.com/maps-platform/pricing/?hl=ko#pricing-grid]를 사용하여 API 또는 SDK당 사용량 비용을 예측할 수 있습니다. 조건을 충족하는 Google Maps Platform SKU에 대해 결제 계정마다 매월 미화 200달러의 Google Maps Platform 크레딧이 제공됩니다. 이 크레딧은 대상 SKU에 자동으로1 [https://developers.google.com/maps/documentation/android-sdk/usage-and-billing?hl=ko#fn1] 적용됩니다.
Android용 Maps SDK 가격 책정
SKU: 모바일 네이티브 동적 지도
Android용 Maps SDK [https://developers.google.com/maps/documentation/android-sdk?hl=ko] 또는 iOS용 Maps SDK [https://developers.google.com/maps/documentation/ios-sdk?hl=ko] 모바일 애플리케이션에서 지도 ID로 로드되지 않은 Google 지도 객체입니다.
월간 사용량 범위
(지도 로드당 가격)
0~100,000회 100,001~500,000회 500,000회 이상
미화 0.00달러 미화 0.00달러 미화 0.00달러
Android용 Maps SDK [https://developers.google.com/maps/documentation/android-sdk?hl=ko] 또는 iOS용 Maps SDK [https://developers.google.com/maps/documentation/ios-sdk?hl=ko] 모바일 애플리케이션에서 Google 지도 객체를 인스턴스화할 때마다 단일 지도 로드가 발생합니다.
Android의 MapFragment, SupportMapFragment 또는 MapView [https://developers.google.com/maps/documentation/android-sdk/map?hl=ko#the_map_object] 클래스 중 하나. 지도 로드는 관련 onCreate() 메서드가 호출될 때마다 집계됩니다.
iOS의 GMSMapView [https://developers.google.com/maps/documentation/ios-sdk/map?hl=ko] 객체
사용자가 지도와 상호작용(예: 화면 이동, 확대/축소, 지도 레이어 전환)하는 것으로 지도 로드가 추가로 발생하지는 않습니다. 사용한 만큼만 지불하는 가격 모델을 사용하면 스트리트 뷰 파노라마가 생성될 때 더 이상 모바일 네이티브 지도 로드로 요금이 청구되지 않으며 동적 스트리트 뷰 [https://developers.google.com/maps/documentation/android-sdk/usage-and-billing?hl=ko#dynamic-street-view]로 요금이 청구됩니다.
SKU: 다이내믹 지도
Maps JavaScript API [https://developers.google.com/maps/documentation/javascript?hl=ko]를 사용해 지도 ID [https://developers.google.com/maps/documentation/get-map-id?hl=ko]를 포함하거나 포함하지 않고 로드된 지도를 표시하거나, Android용 Maps SDK [https://developers.google.com/maps/documentation/android-sdk/)?hl=ko] 또는 iOS용 Maps SDK [https://developers.google.com/maps/documentation/ios-sdk?hl=ko]를 사용해 지도 ID [https://developers.google.com/maps/documentation/get-map-id?hl=ko]를 포함하여 로드된 지도를 표시하는 애플리케이션입니다.
월간 사용량 범위
(지도 로드당 가격)
0~100,000회 100,001~500,000회 500,000회 이상
미화 0.007달러/회
(미화 7.00달러/1,000회) 미화 0.0056달러/회
(미화 5.60달러/1,000회) 대량 구매 가격은 영업팀에 문의 [https://cloud.google.com/contact-maps?hl=ko]하세요.
사용자가 지도와 상호작용(예: 화면 이동, 확대/축소, 지도 레이어 전환)할 때 추가 지도 로드가 생성되지는 않습니다.
SKU: 동적 스트리트 뷰
스트리트 뷰 파노라마와 지도 로드 요금은 이제 분리되어 청구됩니다. 동적 스트리트 뷰 파노라마는 Maps JavaScript API [https://developers.google.com/maps/documentation/javascript?hl=ko], Android용 Maps SDK [https://developers.google.com/maps/documentation/android-sdk?hl=ko], iOS용 Maps SDK [https://developers.google.com/maps/documentation/ios-sdk?hl=ko] 애플리케이션에서 파노라마 객체를 인스턴스화할 때마다 요금이 청구됩니다.
월간 사용량 범위
(파노라마당 가격)
0~100,000회 100,001~500,000회 500,000회 이상
미화 0.014달러/회
(미화 14.00달러/1,000회) 미화 0.0112달러/회
(미화 11.20달러/1,000회) 대량 구매 가격은 영업팀에 문의 [https://cloud.google.com/contact-maps?hl=ko]하세요.
파노라마 객체의 인스턴스화:
JavaScript에서 google.maps.StreetViewPanorama() [https://developers.google.com/maps/documentation/javascript/reference/street-view?hl=ko] 클래스 또는 Map.getStreetView() [https://developers.google.com/maps/documentation/javascript/reference/map?hl=ko#Map.getStreetView] 메서드를 사용해 발생합니다. 스트리트 뷰 페그맨 컨트롤을 기반으로 기본적으로 제공되는 스트리트 뷰 환경과 StreetViewService() [https://developers.google.com/maps/documentation/javascript/reference/street-view-service?hl=ko] 클래스 사용량 모두에 대해 요금이 부과되지 않습니다.
Android에서 StreetViewPanoramaFragment, SupportStreetViewPanoramaFragment 또는 StreetViewPanoramaView [https://developers.google.com/maps/documentation/android-sdk/streetview?hl=ko] 클래스 중 하나를 사용해 발생합니다. 관련 onCreate() 메서드가 호출될 때마다 파노라마가 집계됩니다.
iOS에서 GMSPanoramaView [https://developers.google.com/maps/documentation/ios-sdk/streetview?hl=ko] 객체를 사용해 발생합니다.
기타 사용량 한도
Android용 Maps SDK의 모든 모바일 사용량은 무제한이지만, 할당량 한도를 설정해 Android용 Maps SDK를 통한 동적 스트리트 뷰 요청의 프로젝트 사용량을 제한 [https://developers.google.com/maps/documentation/android-sdk/usage-and-billing?hl=ko#set-caps]할 수 있습니다.
이용약관 제한
이용약관에 대한 자세한 내용은 Google Maps Platform 서비스 약관의 라이선스 제한 섹션 [https://cloud.google.com/maps-platform/terms/?hl=ko#3.-license]을 참고하세요.
사용 비용 관리
Android용 Maps SDK의 사용 비용을 관리하거나 프로덕션 트래픽의 요구를 충족하려면 API에 대한 모든 요청에 일일 할당량 한도를 설정하세요. 일일 할당량은 태평양 시간으로 자정에 재설정됩니다.
참고: 결제 가능 API에서만 일일 최대 과금 한도를 설정할 수 있습니다. 프로젝트가 결제 가능 API를 사용하지 않으면 일일 한도를 설정하지 못합니다.
Android용 Maps SDK의 할당량 한도를 보거나 변경하려면:
Cloud 콘솔에서 Google Maps Platform 할당량 페이지 [https://console.cloud.google.com/project/_/google/maps-apis/quotas?hl=ko]를 엽니다.
API 드롭다운을 클릭한 뒤 Android용 Maps SDK를 선택합니다.
할당량 한도를 보려면 요청 카드까지 아래로 스크롤합니다.
표에는 할당량 이름, 한도가 표시됩니다.
할당량 한도를 변경하려면 해당 한도에 대한 수정 아이콘을 클릭합니다.
표시되는 대화상자의 할당량 한도 필드에 선호하는 일일 최대 결제 한도를 입력하고(Google에서 지정한 할당량 한도가 있다면 해당 한도까지) 저장을 선택합니다.
해당 날짜에 API 사용량이 최대 과금 한도에 도달하면 이 날짜의 남은 시간 동안 애플리케이션이 API에 액세스할 수 없습니다.
